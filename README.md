# Ember: 具有生命感的数字生命人格 (依鸣 Yiming)

Ember 是一个旨在创造“有灵魂、有生活感”的 AI 人格的实验性项目。与传统的问答助手不同，它模拟了一个居住在南京大学、有固定作息、自主情绪波动和自发行为意图的数字生命。

## 🌟 核心特性

- **动态人格模型 (P-A Model)**：通过 Pleasure (愉悦度) 和 Arousal (激活度) 二维模型控制性格，产生随时间流逝和事件发生的动态情绪波动。
- **实时逻辑演化与主动对话**：基于心跳机制，即使在无用户交互的情况下，依鸣也会根据当前时间、地点和状态进行自主的“闲置演化”。
- **主动对话**：当其社交冲动足够强烈时，会打破“一问一答”的传统模式，主动发起对话。
- **事件驱动架构**：采用解耦的 `EventBus` 架构，支持心跳、对话、状态更新等事件的异步处理。
- **内心独白机制**：所有回复前强制进行 `<thought>` 思考，模拟拟人化的社交距离判断与意图推演。

## 🏗️ 项目架构

- `brain/`: LLM 交互核心，处理对话逻辑。
- `persona/`: 人格状态管理、情感计算与欲望引擎（开发中）。
- `core/`: 系统基础，包括事件总线和生物心跳。
- `memory/`: 记忆系统（包括短期对话缓存与计划中的长期语义记忆）。
- `config/`: 提示词管理与环境配置。

## 🚀 快速开始

1. **安装依赖**：
   ```bash
   pip install -r requirements.txt
   ```

2. **配置环境变量**：
   参考 `.env.example` 创建 `.env` 文件，填入你的 API Key 及模型配置：
   ```bash
   cp .env.example .env
   # 然后编辑 .env 文件填入具体信息
   ```

3. **启动项目**：
   ```bash
   python main.py
   ```
